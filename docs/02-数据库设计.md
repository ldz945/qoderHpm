# HPM数据库设计

## 1. 主数据表（来自TOMS接口）

### 1.1 HPM_EMPLOYEE（员工表）

存储从TOMS同步的员工基础信息。

| 字段名 | 字段中文名 | 数据类型 | 是否必填 | 默认值 | 说明/约束 |
|--------|------------|----------|----------|--------|-----------|
| employee_id | 员工ID | NUMBER | 是 | - | 主键，TOMS系统ID，唯一标识 |
| employee_name | 员工全名 | VARCHAR(100) | 是 | - | 员工姓名 |
| employee_code | 员工工号 | VARCHAR(50) | 是 | - | 员工工号，唯一 |
| department | 员工部门 | VARCHAR(100) | 否 | - | 所属部门 |
| email | 邮箱 | VARCHAR(100) | 否 | - | 员工邮箱地址 |
| expiry_date | 失效日期 | DATE | 否 | - | 员工账号失效日期 |
| source | 来源 | VARCHAR(20) | 是 | 'TOMS' | 数据来源：TOMS/HPM |
| is_external | 是否外部员工 | VARCHAR(1) | 是 | 'N' | Y=外部员工，N=内部员工 |
| created_at | 创建时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录更新时间 |

**索引建议：**
- `idx_employee_code`：员工工号索引，用于快速查询
- `idx_employee_name`：员工姓名索引，用于模糊查询
- `idx_department`：部门索引，用于按部门筛选

**唯一性约束：**
- `uk_employee_id`：employee_id 唯一
- `uk_employee_code`：employee_code 唯一

**外键关系：**
- 无外键，为主数据表

---

### 1.2 HPM_RESOURCE（资源表）

存储从TOMS同步的资源信息，HPM可维护部分扩展字段。

| 字段名 | 字段中文名 | 数据类型 | 是否必填 | 默认值 | 说明/约束 |
|--------|------------|----------|----------|--------|-----------|
| resource_pk_id | 资源主键ID | NUMBER | 是 | - | 主键，HPM内部自增 |
| source_id | TOMS系统ID | NUMBER | 否 | - | TOMS系统资源ID |
| source_code | TOMS系统CODE | NUMBER | 否 | - | TOMS系统资源CODE |
| resource_code | 资源代码 | VARCHAR(50) | 是 | - | 唯一标识，如H001 |
| resource_type | 资源类型 | VARCHAR(50) | 是 | - | 人工/实验室/测试间/杂项 |
| resource_name | 资源名称 | VARCHAR(200) | 是 | - | 资源全称 |
| daily_charge | 台架资源按天收费 | VARCHAR(1) | 否 | 'N' | Y=按天收费，N=不按天收费 |
| resource_manager | 资源负责人 | VARCHAR(100) | 否 | - | HPM维护，员工姓名或ID |
| short_name | 资源短名 | VARCHAR(50) | 否 | - | 唯一，HPM维护 |
| enabled | 是否启用 | VARCHAR(1) | 是 | 'Y' | Y=启用，N=禁用 |
| created_at | 创建时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录更新时间 |

**索引建议：**
- `idx_resource_code`：资源代码索引
- `idx_resource_type`：资源类型索引
- `idx_resource_manager`：资源负责人索引

**唯一性约束：**
- `uk_resource_code`：resource_code 唯一
- `uk_resource_short_name`：short_name 唯一（非空时）

**外键关系：**
- 无外键，为主数据表

### 1.3 HPM_PRICE_HEADER（价格表头）

存储资源价格版本信息。

| 字段名 | 字段中文名 | 数据类型 | 是否必填 | 默认值 | 说明/约束 |
|--------|------------|----------|----------|--------|-----------|
| header_id | 表头ID | NUMBER | 是 | - | 主键，自增 |
| version_name | 版本名 | VARCHAR(100) | 是 | - | 唯一，如"2024年价格版本" |
| version_no | 版本号 | NUMBER | 是 | - | 数字版本号，用于排序 |
| currency | 货币 | VARCHAR(10) | 是 | 'CNY' | 币种代码 |
| effective_from | 生效日期 | DATE | 否 | - | 价格版本生效日期 |
| effective_to | 失效日期 | DATE | 否 | - | 价格版本失效日期 |
| status | 状态 | VARCHAR(20) | 是 | 'ACTIVE' | ACTIVE=生效，INACTIVE=失效 |
| created_at | 创建时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录更新时间 |

**索引建议：**
- `idx_version_no`：版本号索引
- `idx_effective_date`：生效/失效日期索引

**唯一性约束：**
- `uk_version_name`：version_name 唯一

**外键关系：**
- 无外键

---

### 1.4 HPM_PRICE_LINE（价格表行）

存储具体资源的价格明细。

| 字段名 | 字段中文名 | 数据类型 | 是否必填 | 默认值 | 说明/约束 |
|--------|------------|----------|----------|--------|-----------|
| line_id | 行ID | NUMBER | 是 | - | 主键，自增 |
| header_id | 表头ID | NUMBER | 是 | - | 外键，关联HPM_PRICE_HEADER |
| resource_type | 资源类型 | VARCHAR(50) | 是 | - | 资源类型 |
| resource_code | 资源代码 | VARCHAR(50) | 是 | - | 关联HPM_RESOURCE |
| price | 价格 | NUMBER(18,2) | 是 | 0 | 资源单价 |
| unit | 单位 | VARCHAR(20) | 是 | - | 价格单位（小时/天/次等） |
| created_at | 创建时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录更新时间 |

**索引建议：**
- `idx_price_line_header`：header_id 索引
- `idx_price_line_resource`：resource_code 索引

**唯一性约束：**
- `uk_price_line`：version_no + resource_code 联合唯一（通过关联header获取version_no）

**外键关系：**
- `fk_price_line_header`：header_id → HPM_PRICE_HEADER(header_id)

| project_type | 项目类型 | VARCHAR(20) | 是 | - | VPI/VPC/CPS/Other |
| quote_type | 报价类型 | VARCHAR(50) | 否 | - | 报价类型 |
| contract_amount | 合同金额 | NUMBER(18,2) | 否 | 0 | 合同总金额 |
| currency | 币种 | VARCHAR(10) | 是 | 'CNY' | 币种 |
| status | 状态 | VARCHAR(20) | 是 | 'DRAFT' | 项目状态 |
| pm | 项目经理 | VARCHAR(100) | 否 | - | 项目经理姓名/ID |
| am | 客户经理 | VARCHAR(100) | 否 | - | 客户经理姓名/ID |
| customer_name | 客户名称 | VARCHAR(200) | 否 | - | 客户名称 |
| quote_date | 报价日期 | DATE | 否 | - | 报价日期 |
| effective_date | 生效日期 | DATE | 否 | - | 项目生效日期 |
| project_level | 项目等级 | VARCHAR(10) | 否 | - | L1/L2/L3/L4 |
| health_status | 项目健康状态 | VARCHAR(10) | 否 | 'GREEN' | 红/黄/蓝/绿 |
| health_remark | 健康备注 | VARCHAR(500) | 否 | - | 健康状态说明 |
| created_at | 创建时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录更新时间 |

**索引建议：**
- `idx_project_code`：项目编码索引
- `idx_project_type`：项目类型索引
- `idx_project_status`：项目状态索引
- `idx_project_pm`：项目经理索引
- `idx_project_level`：项目等级索引

**唯一性约束：**
- `uk_project_code_version`：project_code + version 联合唯一

**外键关系：**
- `fk_project_price_version`：price_version_id → HPM_PRICE_HEADER(header_id)

### 1.8 HPM_ACTUAL_HOUR（实际工时表）

| 字段名 | 字段中文名 | 数据类型 | 是否必填 | 默认值 | 说明/约束 |
|--------|------------|----------|----------|--------|-----------|
| actual_hour_id | 实际工时ID | NUMBER | 是 | - | 主键，自增 |
| project_id | 项目ID | VARCHAR(50) | 是 | - | 外键，关联HPM_PROJECT |
| name | 资源标识 | VARCHAR(100) | 是 | - | 人工/设备:工号, 台架:台架编码 |
| hour_date | 工时日期 | DATE | 是 | - | TOMS测试日期 |
| resource_code | 资源代码 | VARCHAR(50) | 否 | - | 关联的资源代码 |
| task_id | 任务ID | NUMBER | 否 | - | 外键，关联HPM_PROJECT_TASK |
| work_hours | 工时 | NUMBER(10,2) | 否 | 0 | 人工/设备工时 |
| run_time | 运行时间 | NUMBER(10,2) | 否 | 0 | 仅台架：运行时间 |
| stop_time | 停机时间 | NUMBER(10,2) | 否 | 0 | 仅台架：停机时间 |
| sync_month | 同步月份 | VARCHAR(10) | 是 | - | 数据同步月份，如2024-01 |
| created_at | 创建时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录更新时间 |

**索引建议：**
- `idx_actual_project`：project_id 索引
- `idx_actual_date`：hour_date 索引
- `idx_actual_resource`：resource_code 索引
- `idx_actual_sync_month`：sync_month 索引

**唯一性约束：**
- 无（允许同一项目同一资源同一天多条记录）

**外键关系：**
- `fk_actual_project`：project_id → HPM_PROJECT(project_id)
- `fk_actual_task`：task_id → HPM_PROJECT_TASK(task_id)

## 2. 业务表（HPM自身管理）

| project_id | 项目ID | VARCHAR(50) | 是 | - | 外键，关联HPM_PROJECT |
| parent_task_id | 父任务ID | NUMBER | 否 | - | 上级任务ID，支持WBS层级 |
| task_name | 任务名称 | VARCHAR(200) | 是 | - | 任务名称 |
| task_level | 层级数 | NUMBER | 是 | 1 | 任务层级，1为顶层 |
| pre_task_code | 紧前任务编号 | VARCHAR(50) | 否 | - | 前置任务编码 |
| logic_relation | 任务逻辑关系 | VARCHAR(10) | 否 | 'FS' | FS/SF/FF/SS |
| department | 部门 | VARCHAR(100) | 否 | - | 负责部门 |
| task_owner | 任务负责人 | VARCHAR(100) | 否 | - | 负责人姓名/ID |
| authorized_owner | 被授权的任务负责人 | VARCHAR(100) | 否 | - | 授权负责人 |
| planned_start_date | 预定起始时间 | DATE | 否 | - | 计划开始日期 |
| planned_end_date | 预定完成时间 | DATE | 否 | - | 计划完成日期 |
| workload_days | 工作量 | NUMBER(10,2) | 否 | 0 | 工作量(天) |
| task_status | 任务状态 | VARCHAR(20) | 是 | 'NOT_STARTED' | 未开始/进行中/已完成等 |
| has_deliverable | 上传交付物 | VARCHAR(1) | 是 | 'N' | Y=需要上传，N=不需要 |
| is_hour_task | 工时任务 | VARCHAR(1) | 是 | 'N' | Y=是工时任务，N=否 |
| acceptance_item | 验收项 | VARCHAR(500) | 否 | - | 验收标准/内容 |
| remark | 备注 | VARCHAR(1000) | 否 | - | 任务备注 |
| phase | 阶段 | VARCHAR(50) | 否 | - | PreM0-M0/M0-M1/M1-M2等 |
| progress_percent | 进度百分比 | NUMBER(5,2) | 是 | 0 | 0-100 |
| include_weekend | 周末 | VARCHAR(1) | 是 | 'N' | Y=包含周末，N=不包含 |
| version_id | 版本ID | NUMBER | 否 | - | 关联计划版本 |
| created_by | 创建人 | VARCHAR(100) | 是 | - | 创建人 |
| created_at | 创建时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录创建时间 |
| updated_at | 更新时间 | TIMESTAMP | 是 | CURRENT_TIMESTAMP | 记录更新时间 |

**索引建议：**
- `idx_plan_task_project`：project_id 索引
- `idx_plan_task_parent`：parent_task_id 索引
- `idx_plan_task_phase`：phase 索引
- `idx_plan_task_status`：task_status 索引
- `idx_plan_task_version`：version_id 索引

**唯一性约束：**
- 无（同一项目下任务名称可重复）

**外键关系：**
- `fk_plan_task_project`：project_id → HPM_PROJECT(project_id)

### 3.2 主要表关系说明

#### 3.2.1 主数据关系

1. **HPM_EMPLOYEE** - 独立主数据表，存储员工信息
2. **HPM_RESOURCE** - 独立主数据表，存储资源信息
3. **HPM_PRICE_HEADER** 与 **HPM_PRICE_LINE** - 一对多关系，一个价格版本包含多个资源价格明细

#### 3.2.2 项目核心关系

1. **HPM_PROJECT** 为中心表，所有业务表都关联项目
2. **HPM_PROJECT_TASK** - 多对一关联 HPM_PROJECT，存储TOMS同步的任务
3. **HPM_PROJECT_MEMBER** - 多对一关联 HPM_PROJECT，多对一关联 HPM_EMPLOYEE
4. **HPM_PROJECT** 多对一关联 **HPM_PRICE_HEADER**，指定项目使用的价格版本

#### 3.2.3 计划管理关系

1. **HPM_PLAN_TASK** - 多对一关联 HPM_PROJECT，自关联实现WBS层级结构
2. **HPM_PLAN_VERSION** - 多对一关联 HPM_PROJECT，管理计划版本发布
3. **HPM_RESOURCE_PLAN** - 多对一关联 HPM_PROJECT，多对一关联 HPM_PLAN_TASK
4. **HPM_RESOURCE_RESERVE** - 多对一关联 HPM_PROJECT，记录资源预占

#### 3.2.4 变更与验收关系

1. **HPM_CHANGE_ORDER** - 多对一关联 HPM_PROJECT，记录项目变更历史
2. **HPM_ACCEPTANCE** - 多对一关联 HPM_PROJECT，记录项目验收信息
3. **HPM_PROJECT_SCOPE** - 多对一关联 HPM_PROJECT，管理项目范围版本

#### 3.2.5 辅助功能关系

1. **HPM_ISSUE_RISK** - 多对一关联 HPM_PROJECT
2. **HPM_SEVEN_STEP** - 多对一关联 HPM_PROJECT
3. **HPM_VPP** - 多对一关联 HPM_PROJECT
4. **HPM_SCORECARD** - 多对一关联 HPM_PROJECT
5. **HPM_MEETING** - 多对一关联 HPM_PROJECT
6. **HPM_ACTION_ITEM** - 多对一关联 HPM_PROJECT，多对一关联 HPM_MEETING（可选）

#### 3.2.6 文档管理关系

1. **HPM_DOCUMENT** - 多对一关联 HPM_PROJECT，自关联实现目录层级
2. **HPM_DOCUMENT_FILE** - 多对一关联 HPM_DOCUMENT，存储具体文件

#### 3.2.7 财务相关关系

1. **HPM_MISC_EXPENSE** - 多对一关联 HPM_PROJECT，管理杂项费用
2. **HPM_ACTUAL_HOUR** - 多对一关联 HPM_PROJECT，多对一关联 HPM_PROJECT_TASK，来自TOMS同步

# 08-项目变更模块

## 一、变更单信息

### 1.1 基本信息

| 字段 | 说明 |
|------|------|
| 变更单号 | 系统自动生成，唯一标识 |
| 项目编号 | 关联的项目编号 |
| 项目名称 | 关联的项目名称 |
| 变更类型 | 项目范围 / 计划关键路径 |
| 变更状态 | 新建 / 审批中 / 批准 / 驳回 / 关闭 |
| 创建人 | 变更单创建人 |
| 创建时间 | 变更单创建时间 |

### 1.2 编号规则

**变更单号格式**
```
B + 项目号 + 年月 + 3位流水号
```

**示例**：BXXX201908001
- B：变更单标识
- XXX：项目号
- 201908：年月
- 001：流水号

## 二、变更类型详细规则

### 2.1 项目范围变更

#### 适用场景
- 项目需求发生变化
- 项目边界需要调整
- 项目交付物需要增减

#### 填写字段

| 字段 | 说明 | 填写规则 |
|------|------|----------|
| 变更字段 | 具体变更的属性 | 手工选择或输入 |
| 变更影响 | 变更的影响范围 | 下拉选择 |
| 变更前 | 变更前的值 | 系统自动带出 |
| 变更后 | 变更后的值 | 手工填写 |

#### 自动带出规则
- 填写"变更字段"和"变更影响"后
- 系统自动带出"变更前"字段的值

#### L1/L2项目特殊规则
- "变更影响"选择"关键路径"
- 需额外维护：计划变更的层级编码和变更内容

### 2.2 计划关键路径变更

#### 适用场景
- 项目计划需要调整关键路径
- 仅适用于L1/L2项目

#### 填写字段

| 字段 | 说明 | 必填 |
|------|------|------|
| 变更类型 | 计划关键路径 | Y |
| 变更原因 | 变更的背景和原因 | Y |
| 变更内容 | 具体的变更描述 | Y |
| 任务编号 | 受影响的任务编号 | Y |

#### 任务编号限制
- 仅允许填写L1/L2项目中层数为1的任务编号
- 即一级任务（阶段任务）

#### 审批后处理
- 审批通过后
- 对应任务状态自动变为"可修改"
- 提单人/PM可修改计划

## 三、业务流程

### 3.1 项目范围变更流程

```
新建变更单 → 填写变更信息 → 提交审批 → 审批中 → 批准/驳回 → 关闭
                ↓                              ↓
           系统自动带出                    批准：更新项目信息
           变更前值                        驳回：返回修改或关闭
```

### 3.2 计划关键路径变更流程

```
新建变更单 → 选择变更类型 → 填写变更内容 → 提交审批 → 审批中 → 批准/驳回
                                              ↓
                                         批准：任务状态变为"可修改"
                                              ↓
                                         PM修改计划 → 计划变更完成
```

---

## 四、审批规则

### 4.1 审批流程

| 变更类型 | 审批流程 |
|----------|----------|
| 项目范围变更 | 项目经理 → 项目负责人 → 批准/驳回 |
| 计划关键路径变更 | 项目经理 → 项目总监 → 批准/驳回 |

### 4.2 审批动作

| 动作 | 说明 | 后续处理 |
|------|------|----------|
| 批准 | 同意变更 | 执行变更内容，更新相关数据 |
| 驳回 | 不同意变更 | 返回提单人，可修改后重新提交或关闭 |

### 4.3 驳回后处理

| 选项 | 说明 |
|------|------|
| 修改重新提交 | 根据驳回意见修改变更内容，再次提交审批 |
| 手动关闭 | 放弃本次变更，变更单状态变为"关闭" |

## 五、功能点清单

### 5.1 变更单创建
- 选择变更类型
- 填写变更基本信息
- 保存为草稿

### 5.2 变更单编辑
- 修改变更内容
- 查看变更前/后对比
- 添加变更附件

### 5.3 变更单提交
- 校验必填项
- 触发审批流程
- 状态变为"审批中"

### 5.4 变更单审批
- 查看变更详情
- 批准或驳回
- 填写审批意见

### 5.5 变更单查询
- 按项目编号查询
- 按变更类型查询
- 按变更状态查询
- 按时间范围查询

### 5.6 变更单关闭
- 审批通过后自动关闭
- 驳回后手动关闭
- 关闭后不可修改

## 八、校验规则

| 序号 | 校验项 | 规则说明 |
|------|--------|----------|
| 1 | 项目有效性 | 变更的项目必须存在且状态允许变更 |
| 2 | 变更类型必填 | 必须选择变更类型 |
| 3 | 变更内容必填 | 必须填写变更原因和内容 |
| 4 | 任务编号有效性 | 计划关键路径变更必须选择有效的一级任务 |
| 5 | 权限校验 | 只有创建人或管理员可编辑草稿 |
| 6 | 状态流转 | 只有草稿状态可提交，只有审批中状态可审批 |
| 7 | 重复提交 | 同一项目的相同变更类型在审批中时不可再次提交 |

## 九、计算公式

### 9.1 变更单号生成
```
变更单号 = "B" + 项目号 + 年份(4位) + 月份(2位) + 流水号(3位)
```

**示例计算**
- 项目号：XXX
- 当前时间：2019年8月
- 当月第1个变更单：BXXX201908001

### 9.2 流水号规则
- 每月独立计数
- 从001开始递增
- 不足3位前补0

## 十、前置/后置条件

### 10.1 前置条件

| 操作 | 前置条件 |
|------|----------|
| 创建变更单 | 用户已登录且具有创建权限，项目存在 |
| 提交审批 | 变更单状态为"新建"，必填项已填写 |
| 审批变更 | 变更单状态为"审批中"，用户具有审批权限 |
| 修改变更单 | 变更单状态为"新建"或"驳回"，用户为创建人或管理员 |
| 关闭变更单 | 变更单状态不为"关闭" |

### 10.2 后置条件

| 操作 | 后置条件 |
|------|----------|
| 提交审批 | 状态变为"审批中"，通知审批人 |
| 审批批准 | 状态变为"批准"，执行变更内容，通知相关人员 |
| 审批驳回 | 状态变为"驳回"，通知提单人 |
| 关闭变更单 | 状态变为"关闭"，记录关闭原因 |
| 计划关键路径批准 | 对应任务状态变为"可修改"，PM可修改计划 |

# 测试数据说明

## 生成方法

使用以下命令生成测试数据：

```bash
cd backend
python manage.py generate_test_data --projects 5
```

参数说明：
- `--projects`: 生成的项目数量（默认：5）

## 生成的测试数据概览

### 1. 主数据 (Master Data)

#### 员工 (Employee) - 10名
- 员工工号：EMP001 - EMP010
- 部门分布：研发部、测试部、产品部、市场部、质量部、项目部
- 数据来源：HPM

#### 资源 (Resource) - 8个
- 人员资源 (PERSON)：高级工程师、测试工程师、项目经理、产品经理
- 设备资源 (EQUIPMENT)：测试设备A、测试设备B
- 台位资源 (BENCH)：台位1号、台位2号

#### 价格体系 (Price)
- 价格版本：2024标准价格 (v1)
- 货币：CNY
- 价格明细：8条（包含人员、设备、台位的小时/天单价）

### 2. 项目数据 (Project) - 5个项目

每个项目包含以下完整数据：

#### 项目基本信息
- 项目ID：PRJ2026001 - PRJ2026005
- 项目编码：HPM-001 - HPM-005
- 项目名称：测试项目1号 - 测试项目5号
- 项目类型：随机分配 (VPI/VPC/CPS/Other)
- 项目状态：随机分配 (草稿/进行中/暂停/已关闭)
- 合同金额：10万 - 100万随机
- 项目等级：L1 - L4随机
- 健康状态：绿色/蓝色/黄色/红色随机

#### 项目任务 (Project Task) - 每个项目5个
- 需求分析
- 系统设计
- 编码实现
- 测试验证
- 部署上线

#### 项目成员 (Project Member) - 每个项目5个
- 角色：项目经理、开发工程师、测试工程师、产品经理、技术负责人
- 核心成员：前3名

#### 实际工时 (Actual Hour) - 每个项目10条
- 最近10天的工时记录
- 工时：4-8小时随机
- 运行时间：6-10小时随机

#### 计划版本 (Plan Version) - 每个项目1个
- 版本号：1.0
- 状态：已发布

#### 计划任务/WBS (Plan Task) - 每个项目16个
- 4个阶段（父任务）：需求阶段、设计阶段、开发阶段、测试阶段
- 每个阶段3个子任务，共12个子任务
- 任务状态：随机分配

#### 资源计划 (Resource Plan) - 每个项目4个
- 为每个阶段的父任务分配资源
- 资源类型：人员、设备、台位轮换
- 数量：1-3个随机

#### 资源预占 (Resource Reserve) - 每个项目3个
- 预占类型：台位、人员、设备
- 预占状态：已预占
- 预占时长：5天

#### 变更单 (Change Order) - 每个项目2个
- 变更类型：范围变更
- 状态：草稿/待审批/已审批随机

#### 验收单 (Acceptance) - 每个项目2个
- 验收类型：阶段验收
- 状态：草稿/进行中/已完成随机

#### 问题/风险 (Issue/Risk) - 每个项目3个
- 类型：问题/风险随机
- 严重程度：重大影响/一定影响/无影响随机
- 状态：草稿/进行中/已完成随机

#### 七步法 (Seven Step) - 每个项目1个
- 当前步骤：第3步
- 状态：进行中

#### VPP - 每个项目1个
- 版本号：1.0
- 状态：已发布

#### 积分卡 (Scorecard) - 每个项目2个
- 阶段：需求阶段、设计阶段
- 模板类型：L1
- 状态：已完成

#### 会议 (Meeting) - 每个项目3个
- 会议类型：周会
- 会议时长：120分钟
- 状态：已安排

#### 行动项 (Action Item) - 每个项目5个
- 优先级：高/中/低随机
- 状态：进行中/已完成随机
- 计划完成日期：7-30天后

#### 杂项费用 (Misc Expense) - 每个项目4条
- 费用类型：备用金、油耗、项目差旅、采购
- 包含预计和实际金额

#### 项目范围 (Project Scope) - 每个项目1个
- 版本号：1.0
- 状态：生效

#### 文档目录 (Document) - 每个项目9个
- 3个一级目录
- 每个一级目录下2个二级目录，共6个二级目录

#### 文档文件 (Document File) - 每个项目5个
- 文件类型：PDF
- 文件大小：100KB - 1MB随机

## 使用建议

1. **开发测试**：测试数据覆盖了所有主要业务场景
2. **UI演示**：数据丰富，适合前端展示和交互演示
3. **性能测试**：可以通过增加项目数量来测试系统性能
4. **数据重置**：使用 `python manage.py flush --no-input` 清空数据后重新生成

## 注意事项

- 测试数据使用 `get_or_create` 方法，重复执行不会产生重复数据（除了ProjectTask的ID冲突）
- 如需重新生成，建议先执行 `flush` 命令清空数据库
- 所有时间相关数据基于当前日期生成
- 随机数据每次生成可能不同

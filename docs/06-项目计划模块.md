# 06-项目计划模块

## 一、项目计划编制

### 1.1 业务流程

1. 通过项目计划搜索界面查询项目列表（按项目编号、项目经理查询），计划状态为"草稿"
2. 进入更新项目计划界面进行任务编制
3. 创建1级任务（阶段任务），维护阶段时点(M0-M6)
4. 展开编制下级任务
5. 可授权部门人员为"被授权的任务负责人"进行下级操作
6. 维护资源计划（台架、人员、设备）
7. 项目经理检查全部计划后提交审批
8. PL/项目总监审批 → 邮件发布

### 1.2 功能点详细说明

#### 1.2.1 任务创建方式

| 创建方式 | 说明 | 使用场景 |
|----------|------|----------|
| 新建 | 通过"新建"按钮创建首个任务 | 空白项目首次创建 |
| 向下添加 | 通过"任务操作"→"向下添加"创建下级任务 | 创建子任务，层级数自动标识 |
| 平级添加 | 通过"任务操作"→"+"创建平级任务 | 创建同级任务 |
| TBQ导入 | TBQ项目通过"TBQ导入"将立项接收的任务展现 | TBQ项目专用，导入后层级数均为1 |
| 任务计划导入 | 分阶段导入，第一级草稿时可覆盖式导入 | 批量导入任务计划 |
| 资源计划导入 | 第一级草稿时通过任务编号导入下属资源 | 批量导入资源计划 |
| 项目计划复制 | 复制已有项目实现快速创建 | 相似项目快速创建 |
| 任务库 | 提前制定任务库，创建时可选择应用 | 标准化任务快速选用 |
| 项目库 | 历史项目计划导入维护为标准项目库 | 需审批生效后使用 |

#### 1.2.2 任务字段定义

| 字段名称 | 说明 | 必填 | 备注 |
|----------|------|------|------|
| 任务名称 | 任务描述 | Y | - |
| 紧前任务编号 | 前置任务标识 | N | 用于建立任务关系 |
| 任务逻辑关系 | FS/SF/FF/SS | N | 详见下文规则 |
| 部门 | 负责部门 | Y | - |
| 任务负责人 | 主要责任人 | Y | - |
| 被授权的任务负责人 | 可代理操作的人员 | N | 可进行下级任务操作 |
| 预定起始时间 | 计划开始日期 | Y | - |
| 预定完成时间 | 计划结束日期 | Y | 自动计算或手工调整 |
| 任务状态 | 当前状态 | Y | 草稿/计划/进行中/完成等 |
| 上传交付物 | Y/N | Y | 进度100%时必须上传 |
| 工作量 | 天 | Y | 任务工期 |
| 周末 | 是否包含周末 | Y | Y/N |
| 工时任务 | Y/N | N | 标识传递至TOMS的任务 |
| 验收项 | 验收标准 | N | - |
| 备注 | 补充说明 | N | - |

#### 1.2.3 计算公式

**完成时间计算**
```
完成时间 = 起始时间 + 工作量 + 周末（单位天）
```

**说明**：
- 起始时间：任务计划开始日期
- 工作量：任务工期（天）
- 周末：是否跨越周末，Y则加2天，N则不加

#### 1.2.4 阶段规则

| 规则项 | 说明 |
|--------|------|
| 第一层级任务 | 必须填写阶段时段 |
| 非第一级任务 | 默认为空，不可填写 |
| 阶段更改权限 | 仅系统管理员可执行 |
| 阶段值选项 | PreM0-M0, M0-M1, M1-M2, M2-M3, M3-M4, M4-M5, M5-M6, PreM0-M0/M1, M0-M1/M2, M1-M2/M3, M2-M3/M4, M3-M4/M5 |

#### 1.2.5 任务逻辑关系详细规则

**FS（完成-开始）**
- 含义：后续任务开始依赖紧前任务完成
- 联动规则：更改紧前任务结束时间，后续任务开始时间联动变更
- 余量时间：可手工调整
- 约束条件：后续任务开始时间不能早于紧前任务完成时间

**SF（开始-完成）**
- 含义：后续任务完成依赖紧前任务开始
- 联动规则：紧前任务开始时间变动，后续任务完成时间联动变动
- 约束条件：后续任务结束时间不能早于紧前任务开始时间

**FF（完成-完成）**
- 含义：后续任务完成依赖紧前任务完成
- 联动规则：紧前任务完成时间变动，后续任务完成时间联动变动
- 约束条件：后续任务完成时间不能早于紧前任务完成时间

**SS（开始-开始）**
- 含义：后续任务开始依赖紧前任务开始
- 联动规则：紧前任务开始时间变动，后续任务开始时间联动变动
- 约束条件：后续任务开始时间不能早于紧前任务开始时间

#### 1.2.6 工时任务规则

- 需传递至TOMS的任务标识
- 只有标记为工时任务的本任务传递至TOMS，其下子任务不传递
- 资源计划和实际对比时，汇总于此父任务进行对比

#### 1.2.7 被授权的任务负责人规则

- 上级任务维护的"被授权的任务负责人"可进行下级任务操作（增加/删除/修改）
- 可继续指定再下级的"被授权的任务负责人"
- 删除操作只能在草稿阶段执行

## 二、资源计划编制

### 2.1 资源类型与维护规则

| 维度 | 人员 | 设备 | 台架 |
|------|------|------|------|
| 具体信息 | 人员姓名(TOMS接口) | 无(不维护具体设备) | 台架编号(如#1)，已被项目占用的才可选 |
| 资源来源 | TOMS传递；含PM资源、任务负责人资源、其他工程师资源 | TOMS传递 | TOMS传递 |
| 资源数量 | 大于0小于等于1.2；按部门汇总不超被占用数否则需审批 | 大于0的整数 | 默认为1 |
| 每天有效时间 | 系统计算=资源数量×8 | 无 | 单位为小时时手动输入(0-24)；单位为天时不可输入 |
| 价格 | TOMS传递，单位小时 | TOMS传递 | TOMS传递 |
| 计算逻辑 | 使用时间=(结束-起始-周末)×每天有效时间；金额=使用时间×单价 | 类似人员 | 单位天：使用时间=(结束-起始-周末)×资源数量；单位小时：同人员 |

### 2.2 资源维护规则

| 序号 | 规则说明 |
|------|----------|
| 1 | 需先完成任务计划维护后才可维护资源 |
| 2 | 只能维护最末级任务的资源 |
| 3 | 已维护最末级资源后新增下级任务 → 原资源计划无效 |
| 4 | 任务负责人和项目经理作为人员资源存在 |
| 5 | 资源负责人不需要关联资源 |

### 2.3 其他字段

| 字段 | 说明 |
|------|------|
| 周末 | Y/N，标识是否包含周末 |
| 价格版本 | 按任务时间选取对应版本 |
| 资源负责人 | 自动带出 |
| 备注 | 补充说明 |
| 预定起始时间 | 资源使用开始时间 |

### 2.4 资源预定结束时间校验

**人员资源**
- 校验规则：该人员需求时间 + 每时段已使用情况(天) <= 1.2 才可选择成功
- 总数控制：需在预占范围内

**设备资源**
- 选择规则：选择目前可用时段
- 关联逻辑：和资源预占关联

**台架资源**
- 占用维度：按天维度占用，不到小时
- 时段警告：选择不可使用时段时确认警告
- 预占逻辑：需满足预占逻辑
- 显示规则：未被预占的台架灰显

## 三、项目计划审批

### 3.1 审批流程

```
项目经理 → 资源负责人 → 台架指定人员审批 → 项目总监(赵世科)
           ↓
    人员资源→各部门负责人
    设备资源→资源负责人
```

### 3.2 发布规则

| 场景 | 规则 |
|------|------|
| 提交时 | 选择发布截止任务阶段或全部发布 |
| 触发审批 | 状态变为【批准中】 |
| 审批拒绝 | 返回【草稿】状态 |
| 审批通过 | 变为【计划】状态 → 邮件发布 |

### 3.3 发布后状态变更

| 条件 | 结果 |
|------|------|
| 第一级任务开始时间 > 当前时间 | 保持"计划"状态，后续手工触发"进行" |
| 第一级任务开始时间 < 当前时间 | 自动变为"进行中" |
| 分阶段发布 | 截止时段任务为"进行中"，其他阶段保持草稿 |

## 五、项目计划更改

### 5.1 更改流程

1. 项目状态变为"可修改"（PM直接更改或变更管理触发）
2. 可修改状态下不可填写进度
3. 任务状态可单独手动更改
4. 判断是否关键路径变更

### 5.2 关键路径变更判断

**有FS关系**
- 更改紧前/后续任务
- 系统判断完成时间是否超出上级范围
- 超出则警告不可更改

**无FS关系**
- 判断完成时间是否在上级范围内
- 超出则警告

### 5.3 项目级别更改规则

| 项目级别 | 1级任务更改规则 |
|----------|-----------------|
| L1/L2项目 | 必须通过变更管理 |
| L3/L4项目 | PM可直接更改 |

### 5.4 任务变更规则

- 可对任务增加/修改
- 不可删除任务
- 失效可取消

### 5.5 版本升级规则

| 变更类型 | 版本规则 |
|----------|----------|
| 项目状态变更 | 升版 + 邮件发布 |
| 仅任务变更，不影响关键路径 | 不升版，仅邮件通知 |

## 六、资源计划更改

### 6.1 更改条件

- 可修改状态下直接进行
- 需满足资源预占逻辑

### 6.2 更改操作

- 新增资源
- 修改资源
- 失效资源（改状态为"无效"）

### 6.3 各类资源更改规则

| 资源类型 | 更改规则 |
|----------|----------|
| 台架 | 需在预占范围内（台架编号+时段） |
| 人员 | 按部门/项目短码维度，总人天数需在占用范围内 |
| 设备 | 以项目维度总体占用，总设备数需在预占范围内 |

### 7.1 同步时机

- 计划发布后实时同步

## 十、校验规则

| 序号 | 校验项 | 规则 |
|------|--------|------|
| 1 | 任务时间 | 完成时间 >= 起始时间 |
| 2 | FS关系 | 后续开始 >= 紧前完成 |
| 3 | SF关系 | 后续结束 >= 紧前开始 |
| 4 | FF关系 | 后续完成 >= 紧前完成 |
| 5 | SS关系 | 后续开始 >= 紧前开始 |
| 6 | 阶段必填 | 第一级任务必须填写阶段 |
| 7 | 资源数量 | 人员>0且<=1.2，设备>0整数 |
| 8 | 人员预占 | 每时段<=1.2 |
| 9 | 台架预占 | 不可选择已被占用的时段 |
| 10 | 导入限制 | 已发布任务不允许再次导入 |

### 11.1 任务编号

### 11.2 计划版本号
- 初始版本：V1.0

## 十二、前置/后置条件

### 12.1 前置条件

| 操作 | 前置条件 |
|------|----------|
| 编制计划 | 项目已立项且计划状态为"草稿" |
| 维护资源 | 任务计划已维护完成 |
| 提交审批 | 计划编制完成且通过自检 |
| 更改计划 | 项目状态为"可修改" |
| 资源预占 | 资源计划已审批通过 |

### 12.2 后置条件

| 操作 | 后置条件 |
|------|----------|
| 计划发布 | 同步任务至TOMS，邮件通知相关人员 |
| 计划变更 | 更新版本号，同步TOMS，邮件通知 |
| 资源预占 | 更新资源占用状态，影响其他项目选择 |
| 审批通过 | 更新项目状态，触发发布流程 |

# TOMS-HPM系统集成接口技术文档

## 概述

本文档定义了TOMS系统与HPM系统之间的集成接口规范，涵盖数据交换的传输方式、字段定义、校验规则、异常处理及监控方案。

---

## 接口1：员工信息接口

### 基本信息

| 项目 | 内容 |
|------|------|
| **接口编号** | IF-001 |
| **接口名称** | 员工信息接口 |
| **传输方向** | TOMS → HPM |
| **传输方式** | WebService |
| **触发条件** | TOMS系统新增/修改员工信息并生效 |
| **数据传输方式** | 增量传送（含修改数据） |

### 请求字段清单

| 序号 | 字段名 | 数据类型 | 是否必填 | 说明/约束 |
|------|--------|----------|----------|-----------|
| 1 | 员工ID | NUMBER | 是 | 唯一字段，员工唯一标识 |
| 2 | 员工全名 | VARCHAR | 是 | 员工完整姓名 |
| 3 | 员工工号 | VARCHAR | 是 | 唯一字段，员工工号标识 |
| 4 | 员工部门 | VARCHAR | 是 | 所属部门名称 |
| 5 | 邮箱 | VARCHAR | 否 | 员工邮箱地址 |
| 6 | 失效日期 | DATE | 否 | 员工信息失效日期 |

### 校验规则

1. **员工部门校验**：员工部门必须在HPM系统中已存在
2. **唯一性校验**：员工ID和员工工号在HPM正式表有效数据中必须唯一

### 异常处理机制

- 当数据校验失败时，需在TOMS系统中修改数据后，重新提交接口表到正式表的请求

### 特殊说明

- 来源于TOMS的员工数据，HPM系统不做修改
- HPM系统允许创建外部员工信息（非TOMS同步的员工）

## 接口2：资源信息接口

### 基本信息

| 项目 | 内容 |
|------|------|
| **接口编号** | IF-002 |
| **接口名称** | 资源信息接口 |
| **传输方向** | TOMS → HPM |
| **传输方式** | WebService |
| **触发条件** | TOMS系统新增资源信息 |
| **数据传输方式** | 增量传送（含修改数据） |

### 请求字段清单

| 序号 | 字段名 | 数据类型 | 是否必填 | 说明/约束 |
|------|--------|----------|----------|-----------|
| 1 | 主键ID | NUMBER | 是 | TOMS系统的资源ID |
| 2 | 来源CODE | NUMBER | 是 | TOMS系统的资源CODE |
| 3 | 资源代码 | VARCHAR | 是 | 资源唯一标识（如：H001） |
| 4 | 资源类型 | VARCHAR | 是 | 取值：人工/实验室/测试间/杂项 |
| 5 | 资源名称 | VARCHAR | 是 | 资源完整名称 |
| 6 | 台架资源按天收费 | VARCHAR | 否 | 取值：Y/N，仅台架资源有效 |

### 校验规则

1. **资源类型校验**：资源类型必须在HPM系统中已存在
2. **唯一性校验**：资源代码在HPM正式表中必须唯一

### HPM需补充维护字段

以下字段在HPM系统中需要人工补充维护：

| 字段名 | 说明 |
|--------|------|
| 资源负责人 | 负责该资源的人员 |
| 资源短名 | 资源简称，需唯一 |
| 启用状态 | 资源是否启用 |

## 接口7：实际资源接口（冻结工时同步接口）

### 基本信息

| 项目 | 内容 |
|------|------|
| **接口编号** | IF-007 |
| **接口名称** | 冻结工时同步接口 |
| **传输方向** | TOMS → HPM |
| **传输方式** | WebService |
| **触发条件** | TOMS系统定时请求，每月将冻结数据传送 |
| **数据传输方式** | 增量传送，冻结后每月一次，仅新增数据（不存在修改和删除） |

### 请求字段清单

| 序号 | 字段名 | 数据类型 | 是否必填 | 说明/约束 |
|------|--------|----------|----------|-----------|
| 1 | 项目ID | NUMBER | 是 | 关联的项目ID |
| 2 | 名称 | VARCHAR | 是 | 人工/设备：工号；台架：台架编码 |
| 3 | 工时日期 | DATE | 是 | TOMS测试日期 |
| 4 | 资源代码 | VARCHAR | 是 | 资源唯一标识 |
| 5 | 任务ID | NUMBER | 是 | 关联的任务ID |
| 6 | 工时 | NUMBER | 条件必填 | 人工/设备工时（人工/设备类型必填） |
| 7 | 运行时间 | NUMBER | 条件必填 | 仅台架类型必填 |
| 8 | 停机时间 | NUMBER | 条件必填 | 仅台架类型必填 |

### 校验规则

1. **项目存在性校验**：项目ID必须在HPM系统中已存在
2. **任务存在性校验**：任务ID必须在HPM系统中已存在
3. **资源存在性校验**：资源代码必须在HPM系统中已存在

### 界面展示要求

- 按**项目编码+项目名称+任务编码+任务名称+工时日期**维度展示
- 不允许修改已同步的数据

## 接口8：项目更新接口

### 基本信息

| 项目 | 内容 |
|------|------|
| **接口编号** | IF-008 |
| **接口名称** | 项目更新接口 |
| **传输方向** | HPM → TOMS |
| **传输方式** | WebService |
| **触发条件** | HPM中项目计划发布及任务更新时，实时同步 |
| **数据传输方式** | 实时传输，异步处理，增量（仅变更/新增数据） |

### 请求字段清单

| 序号 | 字段名 | 数据类型 | 是否必填 | 说明/约束 |
|------|--------|----------|----------|-----------|
| 1 | ID | NUMBER | 是 | 记录唯一标识 |
| 2 | 项目 | DATE | 是 | 项目编码（注：此处数据类型应为VARCHAR） |
| 3 | 任务ID | VARCHAR | 是 | HPM系统任务主键 |
| 4 | 任务名称 | VARCHAR | 是 | 任务完整名称 |
| 5 | Function | VARCHAR | 是 | 部门名称 |
| 6 | 任务状态 | VARCHAR | 是 | 取值：取消/关闭/进行中 |
| 7 | 阶段 | VARCHAR | 否 | 项目阶段 |
| 8 | 是否收费 | VARCHAR | 是 | 同步需更新TOMS任务是否可报工状态 |
| 9 | 接口传输时间 | DATE | 是 | 数据传输时间戳 |

### 异常处理机制

| 异常场景 | 处理方式 |
|----------|----------|
| 接口同步超时 | 查看接口表传输状态，手工提交重传 |
| 接收系统崩溃 | 查看接口表传输状态，手工提交重传 |
| TOMS处理失败 | 查看HPM异常项目最新同步状态及原因 |

### 重要约束

- **已发变更的项目在收到TOMS处理成功状态前，不允许进行下一次变更**

## 接口9：项目处理结果返回接口

### 基本信息

| 项目 | 内容 |
|------|------|
| **接口编号** | IF-009 |
| **接口名称** | 项目处理结果返回接口 |
| **传输方向** | TOMS → HPM |
| **传输方式** | WebService |
| **触发条件** | TOMS系统项目变更处理程序执行完毕后 |
| **数据传输方式** | 实时同步和更新，增量 |

### 请求字段清单

| 序号 | 字段名 | 数据类型 | 是否必填 | 说明/约束 |
|------|--------|----------|----------|-----------|
| 1 | ID | NUMBER | 是 | 记录唯一标识 |
| 2 | 处理状态 | VARCHAR | 是 | 处理结果状态 |
| 3 | 项目 | VARCHAR | 是 | 项目编码 |
| 4 | 异常原因 | VARCHAR | 否 | 处理失败时的异常说明 |
| 5 | 任务ID | VARCHAR | 是 | 关联的任务ID |
| 6 | 接口传输时间 | DATE | 是 | 数据传输时间戳 |

### 处理逻辑

1. HPM接收到处理结果后，更新项目状态，允许进行下一次变更
2. 处理失败时，需优先处理变更异常

## 接口监控方案

### 监控界面功能

接口监控界面为标准功能模块，支持查询以下信息：

| 查询字段 | 说明 |
|----------|------|
| 服务代码 | WebService服务标识代码 |
| 服务名称 | WebService服务名称 |
| 客户端ID | 调用方系统标识 |
| 请求状态 | 接口请求处理状态（成功/失败/处理中） |

### 监控内容

1. **接口调用日志**：记录每次接口调用的请求时间、响应时间、请求参数、响应结果
2. **异常告警**：接口调用失败时触发告警通知
3. **状态追踪**：实时查看接口数据传输状态
4. **重传机制**：支持失败接口的手工重传操作

### 监控指标

| 指标名称 | 说明 |
|----------|------|
| 接口调用成功率 | 成功调用次数/总调用次数 |
| 平均响应时间 | 接口响应时间的平均值 |
| 异常接口数量 | 处于异常状态的接口数量 |
| 待处理数据量 | 等待同步的数据记录数 |

# 07-项目执行模块

## 一、业务流程

1. PM决定是否授权职能负责人维护进度
2. PM或被授权的职能负责人填写末级任务进度
3. 保存后手动发邮件给相关人员

---

## 二、功能点清单

### 2.1 进度维护授权
- 授权人：项目经理(PM)
- 被授权人：职能负责人
- 授权内容：维护指定任务的进度
- 授权范围：可按任务或部门维度授权

### 2.2 任务进度填写
- 填写对象：末级任务
- 填写人：PM或被授权的职能负责人
- 填写内容：完成百分比、实际开始时间、实际完成时间、备注

### 2.3 进度计算与汇总
- 计算范围：仅"进行中"状态的任务参与计算
- 汇总规则：末级任务向上逐级汇总
- 重新计算：末级任务已有进度时允许新增子任务，系统自动重新计算

### 2.4 交付物管理
- 上传方式：任务负责人上传文件或填写PLM文件路径
- 自动标记：上传后"交付物"字段自动标记为Y
- 外部路径："外部路径"字段维护PLM路径也标记Y

### 2.5 进度预警
- 适用范围：计划工作量>5天且进行中的任务
- 预警计算：自动计算偏差率并标记颜色
- 邮件提醒：红色预警每周一次邮件通知

---

## 三、权限矩阵

| 操作 | 项目负责人 | PM | 技术负责人 | 职能负责人 | 工程师 |
|------|-----------|-----|-----------|-----------|--------|
| 查看所有任务进度 | Y | Y | Y | Y | Y |
| 上传所有交付物 | Y | Y | Y | - | - |
| 上传本部门交付物 | - | - | - | Y | - |
| 上传本人负责任务交付物 | - | - | - | - | Y |
| 填写末级任务进度 | - | Y | - | Y(被授权时) | - |

## 四、页面要求

### 4.1 项目执行主页面
- 左侧：项目任务树形结构
- 右侧：任务执行详情
- 顶部：项目基本信息和总体进度

### 4.2 进度填写区域
- 进度条：可视化拖动或输入百分比
- 时间字段：实际开始时间、实际完成时间
- 备注：文本输入框
- 交付物上传：文件上传组件或路径输入

### 4.3 进度看板
- 甘特图视图：计划vs实际对比
- 列表视图：任务进度明细
- 预警标识：颜色标记预警状态

### 4.4 交付物管理页面
- 列表展示：任务名称、交付物状态、上传时间、操作
- 上传功能：支持多文件上传
- 路径维护：PLM外部路径输入

## 五、校验规则

| 序号 | 校验项 | 规则说明 |
|------|--------|----------|
| 1 | 进度填写状态 | 仅"进行中"状态的任务可填写进度 |
| 2 | 进度值域 | 0% - 100% |
| 3 | 时间逻辑 | 实际完成时间 >= 实际开始时间 |
| 4 | 交付物校验 | 进度100%时如勾选"上传交付物"，校验交付物字段是否为Y，N则不允许保存 |
| 5 | 权限校验 | 只有PM或被授权的职能负责人可填写进度 |
| 6 | 任务级别 | 只能填写末级任务进度 |

## 六、计算公式

### 6.1 任务进度汇总
```
父任务进度 = SUM(子任务进度 × 子任务工作量) / SUM(子任务工作量)
```

### 6.2 项目总体进度
```
项目进度 = SUM(一级任务进度 × 一级任务工作量) / SUM(一级任务工作量)
```

### 6.3 计划进度（用于预警计算）
```
计划进度 = (当前时间 - 计划开始时间) / (计划完成时间 - 计划开始时间) × 100%
```

### 6.4 实际剩余进度
```
实际剩余进度 = 100% - 实际进度
```

### 6.5 计划剩余进度
```
计划剩余进度 = 100% - 计划进度
```

### 6.6 偏差率（预警核心公式）
```
偏差率 = (实际剩余进度 - 计划剩余进度) / 计划剩余进度 × 100%
```

### 6.7 预警颜色阈值

| 颜色 | 偏差率范围 | 说明 |
|------|-----------|------|
| 绿色 | 0% - 20% | 正常范围 |
| 黄色 | 20% - 60% | 需要关注 |
| 红色 | > 60% | 严重延期 |

**注**：阈值通过快码维护，可配置调整

### 7.2 交付物编号
- 生成规则：系统自动生成
- 格式：DEL + 任务编号 + 序号

## 八、前置/后置条件

### 8.1 前置条件

| 操作 | 前置条件 |
|------|----------|
| 填写进度 | 任务状态为"进行中"且用户具有填写权限 |
| 上传交付物 | 任务已存在且用户具有上传权限 |
| 查看进度 | 用户已登录且具有项目查看权限 |
| 授权维护进度 | 用户为项目经理 |
| 触发预警 | 任务计划工作量>5天且状态为"进行中" |

### 8.2 后置条件

| 操作 | 后置条件 |
|------|----------|
| 保存进度 | 重新计算父任务进度，更新项目总体进度 |
| 上传交付物 | 更新任务交付物状态，标记为Y |
| 任务完成(100%) | 校验交付物，更新任务状态为"完成" |
| 发送邮件 | 通知相关人员进度更新 |
| 红色预警触发 | 每周一早上4点发送预警邮件 |

---

## 九、进度预警机制详细说明

### 9.1 预警触发条件
- 任务计划工作量 > 5天
- 任务状态 = "进行中"
- 偏差率超过阈值

### 9.2 预警计算时机
- 实时计算：进度保存时
- 定时计算：每日凌晨计算所有符合条件任务

### 9.3 预警邮件发送规则

| 预警级别 | 发送频率 | 发送时间 | 收件人 |
|----------|----------|----------|--------|
| 红色 | 每周一次 | 周一早上4点 | 任务负责人、职能负责人、PM、技术负责人 |
| 黄色 | 不发送邮件 | - | - |
| 绿色 | 不发送邮件 | - | - |

### 9.4 预警展示
- 任务列表：颜色圆点标识
- 甘特图：任务条颜色标识
- 看板：预警任务单独区域展示

---

## 十、交付物管理详细规则

### 10.1 交付物上传方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 文件上传 | 直接上传附件到系统 | 一般文档 |
| PLM路径 | 填写PLM系统文件路径 | 已存PLM的文档 |

### 10.2 交付物状态规则

| 状态 | 说明 | 变更条件 |
|------|------|----------|
| N | 未上传 | 初始状态 |
| Y | 已上传 | 上传文件或填写外部路径后 |

### 10.3 交付物校验规则
- 任务进度达到100%时，如任务要求上传交付物，则必须已标记为Y才能保存
- 校验时机：进度保存时
- 校验失败提示："该任务要求上传交付物，请先上传"

---

## 十一、邮件通知规则

### 11.1 进度更新邮件
- 触发时机：手动点击"发送邮件"按钮
- 收件人：项目相关人员（可配置）
- 内容：进度更新摘要、任务完成情况

### 11.2 预警邮件
- 触发时机：定时任务（每周一4点）
- 收件人：任务负责人、职能负责人、PM、技术负责人
- 内容：红色预警任务列表、偏差率、建议措施

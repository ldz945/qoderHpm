# HPM项目总览

## 1. 项目背景

康明斯东亚研发中心TOMS系统一期已实现项目立项与资源结算的管理，为项目管理提供了基础支撑。HPM系统在此基础上，实现项目管理的全流程覆盖，形成从项目立项、计划、执行、变更到验收的完整闭环管理。

HPM系统以业务为主线，以辅助管理为支撑，以数据可视化为手段，以系统集成作为数据流转保障，构建完整的项目管理解决方案。

### 2.2 HPM覆盖的功能范围

HPM系统覆盖以下功能领域：

- **项目计划管理**：项目任务计划编制、发布、版本管理
- **项目资源计划管理**：人员、设备、台架资源的计划与预占
- **项目进度管理**：任务进度跟踪、里程碑管理、进度预警
- **项目变更管理**：项目范围变更、计划变更的全流程管理
- **项目验收管理**：项目阶段验收、最终验收管理
- **问题管理**：项目问题记录、跟踪、解决
- **风险管理**：项目风险识别、评估、应对
- **七步法管理**：质量问题解决流程管理
- **VPP管理**：项目价值流程图管理
- **积分卡管理**：项目阶段评审积分卡
- **会议管理**：项目会议记录与跟踪
- **行动项管理**：行动项分配与闭环
- **文档管理**：项目文档分类存储与版本控制
- **看板与报表**：项目数据可视化展示

## 3. 模块划分

### 3.1 业务主线

| 模块 | 功能说明 |
|------|----------|
| 项目立项管理 | 从TOMS同步项目立项信息，维护项目基本信息、价格版本、项目成员 |
| 项目计划管理 | 项目任务分解(WBS)、任务依赖关系、计划版本发布 |
| 项目资源计划管理 | 人员、设备、台架资源的计划安排与预占管理 |
| 项目进度管理 | 任务进度填报、里程碑跟踪、进度预警 |
| 项目变更管理 | 项目范围变更、计划变更的申请、审批、执行 |
| 项目验收管理 | 项目阶段验收、最终验收流程管理 |

### 3.2 辅助功能主线

| 模块 | 功能说明 |
|------|----------|
| 问题管理 | 项目问题记录、分配、跟踪、解决闭环 |
| 风险管理 | 项目风险识别、评估、应对计划制定 |
| 七步法 | 质量问题解决的标准七步法流程 |
| VPP | 项目价值流程图(Value Process Profile)管理 |
| 积分卡 | 项目阶段评审积分卡管理(L1/L2模板) |
| 会议管理 | 项目会议安排、纪要、跟踪 |
| 行动项管理 | 行动项创建、分配、跟踪、完成确认 |
| 文档管理 | 项目文档目录管理、文件上传下载、版本控制 |

### 3.3 数据主线

| 模块 | 功能说明 |
|------|----------|
| 看板 | 项目总览看板、资源看板、进度看板等 |
| 报表 | 项目统计报表、资源利用率报表、财务结算报表等 |

### 3.4 集成主线

| 接口方向 | 数据内容 | 同步方式 |
|----------|----------|----------|
| TOMS → HPM | 员工信息、资源信息、价格信息、项目立项、任务信息、项目成员、实际资源 | 增量传送，实际资源每月一次 |
| HPM → TOMS | 项目更新(任务计划发布/变更时) | 实时同步 |
| TOMS → HPM | 项目处理结果返回 | 实时同步 |

## 5. 术语表

### 5.1 角色术语

| 术语 | 英文全称 | 说明 |
|------|----------|------|
| PM | Project Manager | 项目经理，负责项目的整体管理和协调 |
| AM | Account Manager | 客户经理，负责客户关系维护和商务沟通 |
| PL | Project Leader | 项目负责人，负责项目的技术领导和执行 |

### 5.2 项目类型

| 术语 | 说明 |
|------|------|
| VPI | Value Package Introduction，价值包导入项目 |
| VPC | Value Package Change，价值包变更项目 |
| CPS | Customer Project Support，客户项目支持 |
| Other | 其他类型项目，HPM可创建非业务类型项目 |

### 5.3 系统术语

| 术语 | 说明 |
|------|------|
| TOMS | Test Operation Management System，测试运营管理系统 |
| HPM | Project Management System，项目管理系统 |
| TBQ | To Be Quoted，待报价任务 |

### 5.4 业务术语

| 术语 | 说明 |
|------|------|
| VPP | Value Process Profile，价值流程图 |
| 七步法 | 质量问题解决的标准七步法流程(定义→遏制→根本原因→纠正措施→验证→预防→祝贺) |
| 积分卡 | 项目阶段评审评分卡，用于评估项目各阶段执行情况 |
| 台架 | 测试台架资源，按天收费 |
| 阶段 | 项目里程碑阶段：PreM0-M0, M0-M1, M1-M2, M2-M3, M3-M4, M4-M5, M5-M6等 |

### 5.5 项目等级

| 等级 | 说明 |
|------|------|
| L1 | 一级项目，最高优先级 |
| L2 | 二级项目 |
| L3 | 三级项目 |
| L4 | 四级项目 |

### 5.6 项目角色

| 角色 | 说明 |
|------|------|
| 项目负责人 | 项目整体负责人 |
| 项目经理 | 项目日常管理负责人 |
| 项目技术负责人 | 项目技术方案负责人 |
| 职能部门负责人 | 各职能部门在项目中的负责人 |
| 工程师 | 项目执行工程师 |
| 技师 | 项目执行技师 |

### 5.7 项目健康状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| 红 | Red | 项目存在严重问题，需要立即干预 |
| 黄 | Yellow | 项目存在风险，需要关注 |
| 蓝 | Blue | 项目存在轻微问题 |
| 绿 | Green | 项目正常进行 |

### 5.8 任务逻辑关系

| 关系 | 英文 | 说明 |
|------|------|------|
| FS | Finish to Start | 完成-开始，前任务完成后后任务才能开始 |
| SF | Start to Finish | 开始-完成，前任务开始后后任务才能完成 |
| FF | Finish to Finish | 完成-完成，前任务完成后后任务才能完成 |
| SS | Start to Start | 开始-开始，前任务开始后后任务才能开始 |

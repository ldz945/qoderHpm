# 05-项目立项模块

## 一、业务流程

1. 业务人员线下整理立项信息 → 录入TOMS
2. TOMS立项完成 → 通过接口传给HPM（项目编号+版本号唯一校验）
3. HPM查询立项信息、任务信息、成员信息
4. HPM维护项目成员信息（项目角色等）

---

## 二、功能点清单

### 2.1 项目立项汇总查询
- 支持多条件查询：项目编号、项目名称、项目经理、立项状态、立项时间等
- 列表展示关键字段：项目编号、项目名称、项目经理、合同金额、立项状态、健康状态

### 2.2 项目立项详细查看
- 基础信息：项目编号、项目名称、项目类型、立项时间
- 合同信息：合同金额、价格版本
- 任务信息：关联的任务列表
- 成员信息：项目成员详情

### 2.3 项目任务查询
- 只读展示
- 字段：任务编号、任务名称
- 来源：TOMS接口传递

### 2.4 项目成员查询/维护

#### 查询字段
- 员工姓名
- 部门
- 邮箱
- 工号

#### 维护字段
| 字段 | 说明 | 可选值 |
|------|------|--------|
| 人员来源 | 标识人员归属 | 内部工程师 / 外部工程师 |
| 核心成员 | 标识成员重要性 | Y / N |
| 项目角色 | 成员在项目中的职责 | 项目经理、技术负责人、项目成员、职能负责人、质量工程师、其他 |

### 2.5 非业务类型项目新建
- HPM可独立新建非业务类型项目
- 不走TOMS接口，直接在HPM维护完整信息

### 2.6 项目健康状态管理
- 字段：项目健康状态（红黄蓝绿）
- 字段：健康备注
- 规则：项目关闭时自动变为蓝色

### 2.7 合同金额修改
- 修改方式：走变更流程
- 流程：提交变更申请 → 审批 → 更新立项界面
- 历史版本：变更前版本在变更单中查看

### 2.8 项目经理变更
- 执行人：系统管理员
- 流程：直接更改，不走变更流程

### 2.9 项目成员失效
- 执行人：项目经理
- 流程：直接更改成员状态为失效，不走变更流程

## 三、权限矩阵

| 操作 | PM | AM | 系统管理员 |
|------|----|----|-----------|
| 查询立项信息 | Y | - | Y |
| 维护成员角色等 | Y | - | - |
| 修改合同金额 | Y(走变更) | - | - |
| 变更项目经理 | - | - | Y |
| 失效成员 | Y | - | - |

---

## 四、页面要求

### 4.1 项目立项汇总页面
- 查询条件区域：支持多条件组合查询
- 列表区域：分页展示，支持排序
- 操作列：查看详情、维护成员

### 4.2 项目立项详情页面
- 标签页设计：基础信息、合同信息、任务信息、成员信息
- 基础信息页：展示项目基本信息，健康状态可编辑
- 成员信息页：表格展示，支持增删改查

### 4.3 项目成员维护弹窗
- 表格形式展示成员列表
- 支持批量修改人员来源、核心成员、项目角色
- 支持添加新成员、失效已有成员

---

## 五、校验规则

| 序号 | 校验项 | 规则说明 |
|------|--------|----------|
| 1 | 项目编号唯一性 | 项目编号+版本号组合必须唯一 |
| 2 | TOMS数据保护 | TOMS传送的信息HPM不可修改（除合同金额走变更） |
| 3 | 成员必填项 | 新增成员时，姓名、部门、项目角色为必填 |
| 4 | 健康状态值域 | 仅限：红、黄、蓝、绿 |
| 5 | 合同金额变更 | 必须通过变更流程审批后才能更新 |

---

## 六、计算公式

### 6.1 项目编号唯一校验
```
唯一键 = 项目编号 + 版本号
```

### 6.2 健康状态自动变更
```
IF 项目状态 = "关闭" THEN
    健康状态 = "蓝"
END IF
```

---

## 七、编号规则

### 7.1 项目编号
- 来源：TOMS系统传递
- 格式：按TOMS规则生成

### 7.2 非业务项目编号
- 生成规则：HPM内部自动生成
- 格式：NB + 年月 + 4位流水号（如NB2024010001）

---

## 八、前置/后置条件

### 8.1 前置条件

| 操作 | 前置条件 |
|------|----------|
| 查询立项信息 | 用户已登录且具有查询权限 |
| 维护项目成员 | 项目已立项且用户具有维护权限 |
| 修改合同金额 | 项目状态允许变更且已提交变更申请 |
| 变更项目经理 | 用户为系统管理员 |
| 失效成员 | 用户为项目经理且成员状态为有效 |

### 8.2 后置条件

| 操作 | 后置条件 |
|------|----------|
| TOMS接口接收立项 | 触发项目信息查询更新 |
| 维护成员信息 | 更新成员列表，记录操作日志 |
| 合同金额变更审批通过 | 更新立项界面金额，生成变更记录 |
| 项目经理变更 | 更新项目信息，通知相关人员 |
| 项目关闭 | 健康状态自动变为蓝色 |

---

## 九、关键业务规则补充

### 9.1 TOMS数据同步规则
- TOMS传送的信息HPM不可修改（除合同金额走变更流程）
- 部门、人员来源、核心成员字段可直接在HPM更新

### 9.2 价格版本变更规则
1. TOMS维护新版本报价单 + 生效日期
2. 通过接口传递给HPM
3. 后续结算以新生效日期为节点

### 9.3 项目范围维护
- 立项完成后，PM在项目范围维护界面手工维护
- 立项界面提供跳转链接至项目范围维护

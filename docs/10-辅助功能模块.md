# 辅助功能模块

本文档详细介绍HPM系统中的8个辅助功能模块，包括问题/风险管理、七步法管理、项目文档管理、VPP管理、积分卡管理、会议与行动项管理、杂项费用管理和项目范围管理。

---

## 一、问题/风险管理

### 1.1 业务流程

问题/风险管理模块遵循以下业务流程：

1. **问题录入**：OWNER在问题管理界面录入问题单
2. **信息维护**：维护问题信息并指定负责人，点击"提交"
3. **邮件通知**：系统自动发送邮件给负责人（根据问题状态按月/M节点发送提示）
4. **问题评审**：负责人组织问题评审会议
5. **状态更新**：评审完成后更新问题状态

### 1.2 关键字段

#### 基础信息字段

| 字段名称 | 说明 |
|---------|------|
| 编号 | 项目号 + 年月 + 3位流水（如****201909001） |
| 记录人 | 当前登录用户自动获取 |
| 状态 | 草稿、进行中、取消、完成 |
| 能力(Capability) | Incapable(<60%)、Marginal(60-80%)、Capable(>80%) |
| 重要程度(Importance) | Major Impact、Some Impact、No Impact |
| 成员类型 | 员工、外部员工（外部员工需先在人员信息增加） |

#### 其他字段

- 项目名称
- 项目编码
- 问题描述
- 负责人
- 问题类型
- PROGRAM
- NEXT M REVIEW/DATE
- REVISION DATE

#### 颜色矩阵

系统根据"能力"×"重要程度"两个维度显示颜色底纹，便于快速识别问题优先级。

### 1.3 明细字段

问题明细包含以下字段：

- **COUNTERMEASURES**：对策
- **RESPONSIBILITY**：责任人
- **DUE DATE**：到期日
- **STATE**：状态
- **成员类型**：员工/外部员工
- **邮箱地址**：用于接收通知邮件

### 1.4 核心功能

#### 提交功能
- 系统通过邮箱自动发送邮件给责任人
- 在责任人待办事项中形成待办

#### 首页待办
- 可在首页问题/风险待办中查看待处理问题
- 支持快速跳转处理

#### 更新功能
- 对应负责人可更新本人的到期日、状态、备注
- 支持批量更新

#### 导出功能
- 支持导出到特定格式
- 便于离线分析和汇报

## 二、七步法管理

### 2.1 概述

七步法管理模块用于解决项目管理中的技术问题。目前技术问题录入在MCPASS系统管理，后期HPM也将提供录入界面。

### 2.2 编号规则

**F + 项目号 + 年月 + 3位流水**（如F****201909001）

### 2.3 主要字段

| 字段名称 | 说明 |
|---------|------|
| 项目名称 | 关联项目名称 |
| 项目编码 | 关联项目编码 |
| 技术问题描述 | 问题详细描述（建议500汉字以内，1000字母以内） |
| FIGR Owner | 负责人 |
| FIGR Team | 团队 |
| Platform Impact | 平台影响 |
| Local Specialist | 本地专家 |
| Current Step-S2 | 电流阶跃 |
| NEXT Action | 下一步行动项 |
| RISK ASSESSMENT | 风险评估 |

### 2.4 明细字段

| 字段名称 | 说明 |
|---------|------|
| 过程名称 | 七步法各步骤名称 |
| 描述 | 步骤详细描述 |
| Target Date | 目标日期 |
| Complete Date | 完成日期 |

### 2.5 字段限制

- **描述字段**：建议500汉字以内，1000字母以内
- **包含内容**：标点符号、空格、回车均计入限制

### 2.6 七个步骤详解

#### 步骤1：清晰陈述要解决的问题
- 确定一个主导性问题或坚定的假设
- 要求：具体、有内容、可行动

#### 步骤2：分析问题
- 画逻辑树分解问题
- 将复杂问题拆分为可管理的子问题

#### 步骤3：去掉所有非关键问题
- 聚焦重点问题
- 剔除次要干扰因素

#### 步骤4：制定详细工作计划
- 制定可执行的行动计划
- 明确时间节点和责任人

#### 步骤5：进行关键分析
- 以假设和最终产品为导向
- 深入分析关键问题点

#### 步骤6：综合分析调查结果并建立结论
- 整合各阶段分析结果
- 形成系统性结论

#### 步骤7：陈述来龙去脉
- 完整呈现问题解决过程
- 形成经验沉淀

### 2.7 附件上传

- 可在附件栏位上传相关文档
- 需提供文件路径和名称
- 支持多附件管理

## 三、项目文档管理

### 3.1 文件夹创建规则

#### 自动创建规则

| 项目类型 | 创建时机 | 创建内容 |
|---------|---------|---------|
| TOMS立项项目 | 处理接口数据时 | 自动创建项目层文件夹（目录名称=项目编码，描述=项目描述） |
| HPM立项项目 | 审批完成时 | 自动创建项目层文件夹 |
| VPI项目 | PM操作 | 可选择模板并选择需要的部门创建部门文件夹 |
| 非VPI项目 | PM手动 | 需手动创建部门文件夹 |

#### 子文件夹管理

1. **子任务文件夹**：项目核心成员可创建对应部门下层子任务文件夹
2. **编辑权限**：文件夹创建人可编辑目录名称、目录描述、层级编码、是否启用
3. **删除限制**：创建人可删除空文件夹（存在文件不允许删除）

#### 文件大小限制

- 单文件大小限制：**5M**

### 3.2 文件上传规则

#### 项目执行上传

| 关联情况 | 存放路径 |
|---------|---------|
| 关联任务编码 | 自动按项目+部门+任务编码存放 |
| 无关联 | 按项目+部门存放 |

#### 文件管理上传

1. 输入项目编码
2. 选择存放目录
3. 执行上传

#### 版本管理

- 重复文件名自动升级版本
- 不覆盖原文件
- 保留历史版本记录

### 3.3 权限矩阵

| 角色 | 上传权限 | 下载权限 | 查看权限 | 说明 |
|------|---------|---------|---------|------|
| 项目负责人 | 所有文件夹 | 所有文件夹 | 所有文件夹 | 项目角色 |
| 项目经理 | 所有文件夹 | 所有文件夹 | 所有文件夹 | 项目角色 |
| 技术负责人 | 所有文件夹 | 所有文件夹 | 所有文件夹 | 项目角色 |
| 职能负责人 | 本部门文件夹 | 本部门文件夹 | 本部门文件夹 | - |
| 内部工程师 | 本部门文件夹 | 本部门文件夹 | 本部门文件夹 | 通过标记区分 |
| 外部工程师 | 本部门文件夹 | 无 | 无 | 可申请改为内部 |
| 外部人员 | - | - | 授权后所有 | PM在文件夹处维护登录账户 |
| 管理层 | - | - | 所有项目全部 | 登录账户打标记 |

### 3.4 特殊规则

- **项目关闭**：项目验收后状态为"关闭"时，无法上传新文件，仅可查看和下载已有文件
- **文档修改**：需线下修改后重新上传，不支持在线编辑

## 四、VPP管理

### 4.1 概述

VPP（Vehicle Project Performance）管理模块用于对项目产成品从高关联性、耐久、低成本等方面进行评价，便于后期交付评估。

### 4.2 核心功能

#### 新建功能

- 输入项目编码/名称后建立VPP
- 支持导入功能（仅第一次可使用）
- 自动关联项目基础信息

#### 修改功能

- 修改删除需升级版本
- 修改后在"说明处"备注修改内容
- 主表下层按倒序显示：
  - 版本号
  - 创建人（更新人）
  - 创建时间（更新时间）
  - 历史版本说明

#### 查看功能

- 输入项目编码/名称查询
- 支持多维度筛选

#### 导出功能

提供三种导出方式：

1. **主表导出**：仅导出当前主表数据
2. **版本记录导出**：仅导出版本历史记录
3. **完整导出**：主表+版本记录一起导出

#### 授权功能

- PM可授权其他人操作VPP
- 支持细粒度权限控制

## 五、积分卡管理

### 5.1 概述

积分卡管理模块用于跟进项目进度，从立项到交付各阶段大方向完成度记录，帮助项目团队掌握整体进展状况。

### 5.2 核心功能

#### 新建功能

- 输入项目名称/编码后选择模板
- 支持两种模板类型：

| 模板类型 | 特点 |
|---------|------|
| LEVEL1 | 还需填写阶段信息 |
| LEVEL2 | 三个阶段分为3个TAB页展示 |

#### 编辑功能

- 保存后点击编辑进入数据填写界面
- 支持各阶段完成度录入
- 自动计算总体完成度

#### 导出功能

| 模板类型 | 导出方式 |
|---------|---------|
| LEVEL1 | 直接导出 |
| LEVEL2 | 需选择对应TAB页后导出 |

#### 授权功能

- PM可授权其他人操作积分卡
- 支持查看和编辑权限分离

## 六、会议与行动项管理

### 6.1 会议管理

#### 记录字段

| 字段名称 | 说明 |
|---------|------|
| 会议主题 | 会议标题 |
| 日期时间 | 会议召开时间 |
| 组织人 | 会议组织者 |
| 参加人员 | 参会人员列表 |
| 行动项 | 会议中确定的事后跟进事项 |
| 决定项 | 会议中确定的事项 |

#### 决定项

- 记录会议中确定的重要决策
- 作为项目执行依据

#### 行动项

- 会议中确定的事后跟进事项
- 独立功能模块，可从会议记录链接
- 支持单独管理和跟踪

#### 提交功能

- 会议记录完成后发邮件给所有参与人
- 确保信息同步

#### 导出功能

- 支持会议纪要导出
- 便于存档和分发

#### 附件上传

- 可上传会议纪要文档
- 支持多种文档格式

### 6.2 行动项管理

#### 基础信息

| 字段名称 | 说明 |
|---------|------|
| 编号 | 项目号 + 年月 + 3位流水（如****201909001） |
| 记录人 | 当前登录用户自动获取 |
| 状态 | 进行中、取消、已完成 |

#### 更新权限

- 当前负责人可更新行动项
- PM可更新所有行动项

#### 提交功能

- 提交后自动发邮件给当前负责人
- 形成待办提醒

#### 预警邮件

- **发送时间**：每周一自动发送
- **预警范围**：取红色行动项（当前日期 > 计划完成时间 且 状态为进行中）
- **收件人**：相关负责人和PM

## 七、杂项费用管理

### 7.1 预计金额录入

#### 录入路径

通过项目计划搜索 → 跳转至杂费预计金额界面

#### 字段说明

| 字段名称 | 说明 |
|---------|------|
| 项目编号 | 只读，自动带入 |
| 项目名称 | 只读，自动带入 |
| 结算周期 | 年月格式，如201901 |
| 部门 | 下拉选择 |
| 资源类型 | 下拉选择 |
| 资源名 | 手动输入 |
| 预计价格 | 手动输入 |
| 预计数量 | 手动输入 |
| 预计金额 | 系统计算 = 价格 × 数量 |
| 备注 | 手动输入 |
| 状态 | 默认有效，可改为无效 |
| 杂费类型 | 备用金/油耗/项目差旅/采购 |

### 7.2 实际金额

- **数据来源**：TOMS实际杂项费用通过导入方式进入HPM
- **传递维度**：按类型 + 结算周期维度传递汇总总金额
- **更新频率**：根据业务需求定期同步

### 7.3 对比分析

#### 对比维度

按以下维度展现对比数据：

- 项目
- 杂费类型
- 结算周期

#### 数据规则

- HPM杂费取状态为"有效"的数据汇总
- 实际金额与预计金额进行对比
- 支持差异分析和预警

## 八、项目范围管理

### 8.1 概述

项目范围管理模块用于记录与客户确定的项目实施范围，支持范围变更管理，确保项目交付内容与客户期望一致。

### 8.2 状态定义

| 状态 | 说明 |
|------|------|
| 草稿 | 初始录入状态，可自由编辑 |
| 生效 | 已确认的范围，变更需走流程 |
| 可变更 | 允许进行内容变更的状态 |

### 8.3 核心功能

#### 录入功能

- PM录入项目范围信息
- 支持详细范围描述
- 可关联相关文档

#### 变更流程

1. **变更申请**：生效后变更需在变更流程发起
2. **审批流程**：提交审批后等待审批完成
3. **变更执行**：审批完成后允许变更内容并升级版本

#### 链接功能

- 可链接到对应变更流程
- 便于追溯变更历史

#### Program Scope

- **值列表格式**：序号 + 内容
- **序号规则**：按10/20/30排序
- **插入规则**：可中间插入（如11排在10和20之间）

### 8.4 查询功能

| 查询条件 | 显示结果 |
|---------|---------|
| 版本栏位为空 | 显示最新版本 |
| 全部选项 | 显示所有版本，按降序排列 |

### 8.5 导出功能

- 支持范围信息导出
- 包含版本历史记录
- 便于客户确认和存档
